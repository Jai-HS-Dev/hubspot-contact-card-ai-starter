# Start Here

Use this repo to plan a HubSpot contact card before writing implementation code.

## Step 1: Describe The Card

Write:

```text
This contact card helps a HubSpot user see or do ______.
```

## Step 2: Identify Data Sources

Possible sources:

- HubSpot contact properties
- HubSpot associations
- an external CRM or system
- an internal API
- a computed status from a backend endpoint

## Step 3: Choose Auth

If the card needs private data or external APIs, plan the backend and auth carefully.

See `docs/auth-options.md`.

## Step 4: Build The Smallest Useful Card

Start with one card, one clear user action or display, and placeholder data. Add real data only after the structure is safe.
